# 设计与使用数学任务

**来源**: designing&using math tasks.pdf (20 页)
**主题**: 数学任务的设计原则与课堂使用策略

---

## 1. 核心论点与论证逻辑

本文的核心论点是：数学任务是塑造学生数学学习体验的最重要因素，但任务的价值不仅取决于其设计质量，更取决于教师如何在课堂中实施和管理这些任务。作者通过两条论证线索来支持这一观点。

第一条线索关注任务设计：什么样的任务能够促进深层数学学习？作者认为，好的任务需要在数学内容的丰富性、学生的可及性、以及认知挑战性之间取得平衡。任务应该引发学生的数学思考，而不仅仅是技能操练。

第二条线索关注课堂实施：即使是精心设计的任务，如果教师在实施中处理不当（例如过早给出答案、将开放性任务转为封闭性任务），其学习潜力也会大打折扣。因此，任务设计与任务实施必须作为一个整体来考虑。

作者特别强调了任务使用中的"维持认知需求"问题。研究表明，在实际课堂中，高认知需求任务经常被"降格"为低认知需求任务，而反向提升则很少发生。

## 2. 关键概念定义与解释

### 任务的开放性与封闭性
- **封闭性任务 (Closed tasks)**: 有唯一确定的答案和解题路径。如"计算 25 乘以 8"。
- **开放中间型任务 (Open-middle tasks)**: 有确定的答案但允许多种解题路径。如"用任意方法计算一个不规则图形的面积"。
- **开放结果型任务 (Open-ended tasks)**: 允许多种答案。如"设计一个面积为 24 平方厘米的图形"。

### 任务的真实性 (Authenticity)
指任务与现实生活的联系程度。作者区分了三种层次：
- **纯数学任务**: 完全在数学内部，如证明题
- **伪情境任务**: 表面上有生活情境但情境是装饰性的，去掉情境不影响解题
- **真实情境任务**: 情境是任务不可分割的一部分，解题需要考虑情境因素

### 数学任务的"生命周期"
从构思到课堂实施，一个任务经历多个阶段：
1. 任务的原始设计意图
2. 教师对任务的理解和改编
3. 课堂中任务的呈现方式
4. 学生实际经历的任务
5. 学生从任务中获得的学习

每个阶段之间都存在"转化"，可能改变任务的本质。

### 任务的数学深度
指任务能够触及的数学核心程度。浅层任务仅涉及表面的计算或操作，深层任务能够揭示数学结构和关系。

## 3. 具体例子与案例

### 面积任务的不同版本
同一个数学内容"面积"可以设计出认知需求截然不同的任务：
- 低需求版本：计算一个长 5 厘米、宽 3 厘米的长方形的面积
- 高需求版本：给定 12 个小正方形，用它们拼出不同形状的图形，记录每种图形的面积和周长，你发现了什么规律？

### 教师实施的对比案例
两位教师使用同一个探究型任务（探索三角形内角和）。教师 A 让学生自由探索，但在学生汇报时引导他们关注不同方法之间的数学联系。教师 B 在学生开始探索前就给出了"内角和等于 180 度"的结论，让学生去验证。同一个任务，前者维持了高认知需求，后者将其降为验证性活动。

### 任务改编的例子
将教科书中的标准练习"解方程 2x + 3 = 11"改编为更有认知需求的版本："找出三个不同的方程，使得它们的解都是 x = 4。你的方程之间有什么关系？"

## 4. 框架/模型的完整描述

### 任务设计与实施的整合框架

**设计阶段**:
1. **明确数学学习目标**: 学生应该学到什么数学知识、技能或思维方式？
2. **分析学生起点**: 学生已经知道什么？可能有什么误解？
3. **选择任务类型**: 根据目标和学生起点，决定任务的开放程度、情境化程度、认知需求水平
4. **预测学生反应**: 学生可能会用什么策略？可能在哪里卡住？
5. **规划教师行动**: 准备追问、提示、扩展等应对策略

**实施阶段**:
1. **任务呈现 (Launch)**: 清晰呈现任务，确保学生理解要做什么，但不暗示解法
2. **学生工作 (Explore)**: 学生独立或小组工作，教师巡视观察，但不过早干预
3. **全班讨论 (Discuss/Summarize)**: 有选择地分享学生的解法，引导讨论关键的数学想法

**维持认知需求的策略**:
- 当学生求助时，回答问题而不是给出方法
- 鼓励学生互相解释和质疑
- 让学生在困难中停留足够长的时间
- 追问学生的理由和思考过程

## 5. 对数学教学设计的具体启示

- **任务选择要有目的性**: 每一个课堂任务都应该服务于明确的数学学习目标。避免使用"看起来有趣但数学含量低"的活动。
- **预测学生反应是备课的关键环节**: 在使用一个任务之前，教师应该自己先做一遍，然后设想不同水平的学生会如何反应，并准备相应的教学应对。
- **课堂讨论不等于逐个展示**: 全班讨论的价值在于建立不同解法之间的数学联系，引导学生看到更深层的数学结构。教师需要有策略地选择哪些学生的工作来分享，以及分享的顺序。
- **注意任务实施中的"降格"现象**: 教师要有意识地监控自己是否在不经意间降低了任务的认知需求。常见的降格行为包括：过早给提示、将开放问题变成一步步引导、只关注答案不关注过程。
- **任务序列比单个任务更重要**: 持续使用一系列高认知需求的任务，比偶尔使用一个亮眼的任务更能促进学生数学思维的发展。
